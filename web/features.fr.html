fonctionnalités de RPMREBUILD

resumé

rpmrebuild permet de reconstruire un package rpm à partir d'un autre rpm
notes :
  • les anciennes version de rpm fournissaient l'option '--repackage', mais elle a été supprimée depuis quelques années.
  • Sur debian/ubuntu, la commande équivalente est dpkg-repack.
  • Vous pouvez trouver des exemples d'utilisation sur la page usage

fonctions de base

  • depuis la version 0.5, il devrait fonctionner sur n'importe quelle distribution basée sur rpm, cat il ne necessite que peu dependances (bash, awk).
  • en utilisation de base, rpmrebuild ne nécessite aucune connaissance sur la fabrication des rpm. Il suffit de taper rpmrebuild package_name

fonctions avancées

Elles nécessitent quelques connaissances sur les fichiers spec.
Vous pouuvez fabriquer un fichier rpm à partir de :
  • d'un package rpm installé
  • d'un autre fichier rpm, avec l'option -p
Rpmrebuild travaille en 2 étapes :
  1. d'abord il fabrique un fichier spec
  2. ensuite il utilise la commande rpmbuild pour générer le fichier rpm
Le principal interet est de modifier le fichier spec, pour construire un package rpm different.

interactif

Vous pouvez editer et modifier le fichier spec "au vol" avec la commande : rpmrebuild -e package_name

batch

les Plugins permettent d'automatiser les changements sur un fichier spec.
